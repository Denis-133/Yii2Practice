2) Вложенные контроллеры приложения , что это ? 
Создаем в папке controllers папку admin -> controllers/admin -> создаем гласс UserController.php который будет отвечать за работу
с пользователями. 
Прим.
class UserController extends AppController
{
 public function actionIndex(){
     $hi = "Hello Admin";
     return $this->render('index', compact('hi'));
 }
}

Чтобы получить доступ к этому контроллеру в представлении нужно в адресной строке прописать url вида:
site/admin/user/index

Очень важно на этом этапе следить за неймспейсами!
Index - дефолтный action для отображения через адресную строку

Урок 10. Работа с формами и базами данных.
Для работы с формами нужно сделать модели и унаследоваться от двух разных классов.
1) Если нам не нужна работа с базами данных(прим. отправка данных на почту, обратная связь, рассчет чего либо без внесения в базу) ->
расширяем класс Model. Класс Model - базовый класс.
2) Второй класс ActiveRecord, предназначен для работы с базой данных. 

Форма, отображаемая на стороне клиента, в большинстве случаев будет иметь соответствующую модель, 
которая используется для проверки ввода на стороне сервера 
(см. Раздел « Проверка ввода » для получения дополнительных сведений о проверке). 
При создании форм на основе моделей первым шагом является определение самой модели. 
Модель может быть основана либо на классе Active Record , представляющем некоторые данные из базы данных, 
либо на универсальном классе Model (расширяющемся от yii \ base \ Model ) для захвата произвольных входных данных, например, формы входа в систему.

Совет: если поля формы отличаются от столбцов базы данных или форматирование и логика специфичны только для этой формы, 
предпочтительнее создать отдельную модель, расширенную из yii \ base \ Model .
